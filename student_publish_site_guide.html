🌐 Публикация сайта

Как создать репозиторий и опубликовать HTML-сайт бесплатно

Эта инструкция поможет создать собственный репозиторий на GitHub, загрузить туда HTML-файлы и опубликовать сайт бесплатно через GitHub Pages. Такой способ подойдёт для учебных работ, портфолио и небольших проектов.

📦 Что понадобится

✅ Минимум для старта

  • аккаунт на GitHub;
  • готовые HTML-файлы;
  • браузер и доступ в интернет;
  • немного терпения и аккуратности.

  • собственный онлайн-адрес сайта;
  • рабочая HTML-страница, которую можно открыть с любого устройства;
  • возможность сделать QR-код на свой проект.

📋 Пошаговая схема публикации

1

Создайте аккаунт на GitHub

Перейдите на GitHub, зарегистрируйтесь и подтвердите адрес электронной почты. После входа в аккаунт вы сможете создавать собственные репозитории.

2

Создайте новый репозиторий

Нажмите кнопку New repository. Укажите имя репозитория, например my-html-site. Выберите Public, чтобы сайт можно было опубликовать бесплатно и открыть по ссылке.

3

Загрузите HTML-файлы

Откройте репозиторий, нажмите Add file → Upload files и загрузите все нужные HTML-страницы. Лучше, чтобы они лежали в одной папке и были связаны относительными ссылками.

4

Проверьте главный файл сайта

Для корректного открытия корневой страницы сайта главный файл лучше назвать index.html. Если главная страница сейчас называется иначе, сделайте копию или переименуйте её.

5

Включите GitHub Pages

Откройте Settings → Pages. В разделе публикации выберите Deploy from a branch, затем ветку main и папку / (root). После сохранения GitHub опубликует сайт.

6

Подождите публикацию и проверьте адрес

Через несколько минут GitHub Pages создаст публичную ссылку. Откройте её и убедитесь, что сайт отображается без ошибки 404.

7

Используйте ссылку в QR-коде

Если сайт открылся, именно эту ссылку можно использовать для QR-кода, отправки преподавателю или публикации в учебной системе.

📁 Как должна выглядеть структура файлов

Если у вас несколько страниц, все основные HTML-файлы лучше держать рядом. Тогда кнопки и ссылки между ними будут работать корректно.

index.html
student_checklist_interactive.html
student_html_guide.html
student_report_template.html
student_infrastructure_project.html
images/
  photo1.jpg
  photo2.jpg
💡 Совет: если вы добавляете изображения, удобно хранить их в отдельной папке images.

⚠️ Типичные ошибки

❌ Что часто ломает публикацию

  • загрузка ZIP-файла вместо распакованных HTML-страниц;
  • отсутствие файла index.html;
  • ссылки между страницами ведут на файлы, которых нет рядом;
  • репозиторий сделан приватным;
  • сайт ещё не успел опубликоваться, а ссылка проверяется слишком рано.

✅ Как действовать спокойно

  • проверить, что HTML-файлы лежат в репозитории по отдельности;
  • убедиться, что главный файл — index.html;
  • посмотреть, что в настройках Pages выбрана ветка main и папка / (root);
  • подождать несколько минут и обновить страницу.
⚠️ Важно: если сайт открывается по адресу репозитория, но выдаёт 404, почти всегда сначала стоит проверить наличие index.html.
